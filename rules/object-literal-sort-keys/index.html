--- ruleName: object-literal-sort-keys description: |- Checks ordering of keys in object literals. When using the default alphabetical ordering, additional blank lines may be used to group object properties together while keeping the elements within each group in alphabetical order. rationale: Useful in preventing merge conflicts optionsDescription: | By default, this rule checks that keys are in alphabetical order.
